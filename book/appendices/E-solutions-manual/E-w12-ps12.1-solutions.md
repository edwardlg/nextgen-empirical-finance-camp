The CI arithmetic, verified:

```python
# ci_consistency_check.py — verified
coef, se = -1.41, 0.31
ci_low = coef - 1.96 * se
ci_high = coef + 1.96 * se
print(f"CI = ({-ci_high:.2f}, {-ci_low:.2f}) in absolute units")

# Abstract reports (0.8, 2.0) — correct one-decimal rounding.
```

The 23% relative magnitude, verified:

```python
# share_consistency_check.py — verified
pre_treatment_gap = 6.1
effect = 1.4
share = effect / pre_treatment_gap
print(f"{share:.4f}, {round(share*100):d}%")
# 0.2295, 23%
```

The arithmetic round-trips: a coefficient of −1.41 with SE 0.31 implies a 95% CI of (−2.02, −0.80); the abstract rounds the absolute-value CI to (0.8, 2.0); the relative magnitude 1.4/6.1 = 0.2295 ≈ 23%. All five number locations are consistent.

## Common student errors

1. **Filename does not satisfy the convention.** Students use spaces, uppercase, accents, or special characters. The fix is to literally `rename` the file to match `^SYSJ-[A-Z][a-z]+-\d{8}-[a-z0-9-]{1,25}\.pdf$`.
2. **Cover letter is over 400 words.** Most common cause: paragraph 2 (contribution) is rewritten as a mini-abstract. Fix: cut paragraph 2 to two sentences.
3. **CRediT statement uses "the authors contributed equally."** This is not CRediT-compliant. Fix: name specific roles per author.
4. **Manifest has more or fewer files than the packet.** The integrity check fails. Fix: list files first, then write the manifest from the listing.
5. **The headline number disagrees between abstract and Table 2.** The most expensive error — surfaces inconsistency a careful referee will catch. Fix: run the `pdftotext`+`grep` audit *before* assembly.
