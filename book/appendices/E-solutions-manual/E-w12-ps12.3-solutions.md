### 2. License decisions (Part 2)

Four artifacts, four license decisions:

- **Manuscript text:** Creative Commons Attribution 4.0 International (CC BY 4.0). Rationale: the SSRN-recommended preprint license, compatible with most journal CTAs (the journal gets the typeset version under its CTA; the preprint stays CC BY 4.0).
- **Code (Python, SQL, shell scripts):** MIT License. Rationale: permissive open-source license; commercial re-use permitted; well-understood by replicators.
- **Derived data (county-panel parquet files):** CC0 (public-domain dedication). Rationale: the derived data are aggregated to the county-year level, contain no PII (the input FEMA flood maps and NFIP claims data are themselves public-domain or aggregated), and the public-domain dedication maximizes downstream reuse.
- **Walkthrough video:** CC BY 4.0 (same as manuscript). Rationale: consistency with the manuscript text.

Embargo: **None.** The deposit is open from day one. Documented rationale in `metadata.md`: "the camp encourages open public-record dissemination at the moment of deposit; no priority-protection argument applies because the SSRN posting (PS 12.2) already established priority at v1.0.0."

**The `reproduce.sh` master script** (the operational contract):

set -euo pipefail

python scripts/estimate_heterogeneity.py

## A subtle point — the BUILD-MANIFEST as a drift detector

The BUILD-MANIFEST.md is more than documentation; it is the drift detector. If a future replicator runs `reproduce.sh` and computes SHA-256 hashes that *do not* match the manifest, one of three things has happened:

1. **Source code drift.** Someone modified a script in `scripts/` without bumping the version or updating the manifest. The fix is to identify the modification and either roll it back or update the manifest *and* the version.
2. **Dependency drift.** A package (pandas, statsmodels, etc.) was upgraded between the manifest's build time and the replicator's build time, producing slightly different numerical output. The fix is to pin the dependency more tightly (use exact versions in `environment.yml`, not `>=`) or to switch to the Docker fallback, which freezes every dependency.
3. **Environment drift.** A different operating system, a different Python build, a different hardware FPU. The fix is the Docker fallback.

A verified BUILD-MANIFEST mismatch is *informative*: it tells the replicator something has changed. A package without a BUILD-MANIFEST is uncheckable; a package with one is auditable. This is the discipline that separates a reproducible package from a runnable one.

# manifest_drift_check.py — verified, illustrative

## Common student errors

1. **Some Dublin Core elements left empty.** An empty `dc:coverage` or `dc:relation` is a discoverability hole. Fill every element.
2. **One license for everything (CC BY 4.0 for code and data).** Code wants MIT (permissive open-source); data wants CC0 (public-domain dedication for the aggregated derived data). Separate.
3. **The replication script has manual steps.** "Step 1: open the FEMA portal in a browser, log in, click Download, save the file to data/raw/" — this is not reproducible. Replace with `requests.get` or `curl` in `fetch_fema_flood_maps.py`.
4. **The walkthrough video is unedited and 40 minutes long.** Cut to 8–12 minutes; the timestamped transcript is the search index for the longer details.
5. **The deposit memo has no timeline.** Without forward-looking dates, you have no way to know whether the deposit succeeded.
