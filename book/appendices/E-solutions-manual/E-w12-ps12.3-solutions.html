A subtle point — the BUILD-MANIFEST as a drift detector

The BUILD-MANIFEST.md is more than documentation; it is the drift detector. If a future replicator runs reproduce.sh and computes SHA-256 hashes that do not match the manifest, one of three things has happened:

  1. Source code drift. Someone modified a script in scripts/ without bumping the version or updating the manifest. The fix is to identify the modification and either roll it back or update the manifest and the version.
  2. Dependency drift. A package (pandas, statsmodels, etc.) was upgraded between the manifest’s build time and the replicator’s build time, producing slightly different numerical output. The fix is to pin the dependency more tightly (use exact versions in environment.yml, not >=) or to switch to the Docker fallback, which freezes every dependency.
  3. Environment drift. A different operating system, a different Python build, a different hardware FPU. The fix is the Docker fallback.

A verified BUILD-MANIFEST mismatch is informative: it tells the replicator something has changed. A package without a BUILD-MANIFEST is uncheckable; a package with one is auditable. This is the discipline that separates a reproducible package from a runnable one.

## A subtle point — the BUILD-MANIFEST as a drift detector

The BUILD-MANIFEST.md is more than documentation; it is the drift detector. If a future replicator runs `reproduce.sh` and computes SHA-256 hashes that *do not* match the manifest, one of three things has happened:

1. **Source code drift.** Someone modified a script in `scripts/` without bumping the version or updating the manifest. The fix is to identify the modification and either roll it back or update the manifest *and* the version.
2. **Dependency drift.** A package (pandas, statsmodels, etc.) was upgraded between the manifest's build time and the replicator's build time, producing slightly different numerical output. The fix is to pin the dependency more tightly (use exact versions in `environment.yml`, not `>=`) or to switch to the Docker fallback, which freezes every dependency.
3. **Environment drift.** A different operating system, a different Python build, a different hardware FPU. The fix is the Docker fallback.

A verified BUILD-MANIFEST mismatch is *informative*: it tells the replicator something has changed. A package without a BUILD-MANIFEST is uncheckable; a package with one is auditable. This is the discipline that separates a reproducible package from a runnable one.
