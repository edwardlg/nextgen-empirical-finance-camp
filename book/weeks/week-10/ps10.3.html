PS 10.3 — Estimate One Pre-Registered CATE for Your Paper Using a Causal Forest; Report Honest Best-Linear-Projection

A boundary worth holding before you start. PS 10.3 is not “run a causal forest and report whatever it says.” A causal forest will produce a CATE estimate on any dataset, including a dataset where the true CATE is identically zero — and the visual artifacts (the CATE histogram, the CATE-vs-X scatter) will look like real heterogeneity even when there isn’t any, because the forest is a flexible function approximator that finds structure in noise. The discipline that makes the analysis honest is pre-registration of the moderators, sample-splitting for honest inference, and the best-linear-projection diagnostic that tests whether the apparent heterogeneity is real. Skip any of those three and you have produced an artifact, not evidence. This problem set teaches all three in the context of one CATE on your own paper.

Part 1 — The pre-analysis-plan extract (20 points)

The single sin this part prevents: letting the data tell you which moderators to test, then dressing the answer up as if you had pre-specified them. A causal forest’s value comes from the discipline imposed before it sees the data, not from the forest itself.

(a) Name the moderator list (8 pts). Write, in pap-extract.md, the bulleted list of moderators \mathbf{X}_i — the covariates the forest is allowed to use to predict \tau_i. The list should have between 3 and 12 moderators. Fewer than 3 and the forest cannot find heterogeneity; more than 12 and the forest will overfit and the BLP will lose power (recall the multiple-testing problem of Ch 10.2 returns at every leaf split). Each moderator gets a one-sentence justification of the form: “X_j is included because [theoretical / institutional reason connecting X_j to the mechanism].” For Maya, the four moderators are intensity (FFIEC exam scope), county minority share (decennial census), MSA indicator (OMB definition), CRA-evaluation-period overlap (FFIEC CRA calendar). For Leah, the six moderators are industry FE (SIC 2-digit), market cap (CRSP), R&D intensity (Compustat XRD/AT), patent-stock age (USPTO), citation count (USPTO + Bessen), and litigation history (Stanford NPE database). State your six.

(b) Outcome, treatment, sample (3 pts). Restate the outcome variable, the treatment variable, the sample (same as your headline regression — do not expand the sample for CATE, that is a fork), the time window, and the unit of analysis (county-year, firm-year, etc.). Three sentences total. This is the spec the forest will be fit on; locking it here prevents drift.

(c) Cross-validation / sample-splitting protocol (5 pts). State the honesty protocol you will use. The default this textbook recommends, following Athey-Wager (2019):

  • 5-fold cross-fitting: each unit’s \hat\tau(\mathbf{x}_i) is estimated from a forest trained on a fold that does not contain that unit. This is the property that makes the BLP standard errors valid.
  • Within each fold, honest sample-splitting at the leaf level: each tree splits the training data into two random halves; one half determines tree structure (the splits), the other half estimates the leaf-mean \hat\tau at each leaf. This is the property that makes the per-unit \hat\tau(\mathbf{x}_i) asymptotically normal.
  • Clustering at the cluster level your headline regression used: cross-fitting folds are formed by whole clusters, not units, so the same cluster never appears in both train and test. For panel data, this is usually whole firms or whole counties (not firm-years).

Write this protocol as three sentences. It is the part the forest’s subforest_size and honest=True arguments in econml (or honesty=TRUE in grf) implement; the protocol-as-prose is what your PAP-extract documents.

(d) Lock the PAP-extract and tag the commit (4 pts). Commit cate/pap-extract.md to the repository with the message cate: PAP-extract locked, moderators pre-specified, then tag the commit cate-pap-locked. From this commit forward, the moderator list is frozen — adding a moderator after the forest has produced an interesting CATE-vs-X plot is fishing, and is detectable by git log -- cate/pap-extract.md. If you must add a moderator (because the forest exposed a feature you had not considered), do it transparently: a new commit, a new tag (cate-pap-revised), and a paragraph in the memo (Part 6) explaining why. The transparency makes the revision honest; the silent edit is the dishonesty.

Part 2 — Fit the causal forest with honest sample-splitting (20 points)

(a) Implement with econml.dml.CausalForestDML (10 pts). The script cate/forest.py looks like this skeleton:

# cate/forest.py — causal forest with honest sample-splitting and cross-fitting
import numpy as np
import pandas as pd
from econml.dml import CausalForestDML
from sklearn.linear_model import LassoCV
from sklearn.ensemble import RandomForestRegressor, RandomForestClassifier
from config import SEED

rng = np.random.default_rng(SEED)
df = pd.read_parquet("data/processed/analysis.parquet")

# Moderators X (the heterogeneity-driving features), treatment T, outcome Y,
# nuisance controls W (all the controls your headline regression had).
X_cols = ["intensity_bin", "minority_share", "msa", "cra_overlap"]
W_cols = ["unemp", "median_income", "share_white", "median_age"]  # confounders
T_col, Y_col = "treat_post", "denial_gap"
cluster_col = "county_fips"

X = df[X_cols].to_numpy()
W = df[W_cols].to_numpy()
T = df[T_col].to_numpy()
Y = df[Y_col].to_numpy()
clusters = df[cluster_col].to_numpy()

forest = CausalForestDML(
    model_t=RandomForestClassifier(n_estimators=200, random_state=SEED, n_jobs=-1),
    model_y=RandomForestRegressor(n_estimators=200, random_state=SEED, n_jobs=-1),
    n_estimators=2000,                      # trees in the causal forest itself
    min_samples_leaf=50,                    # leaf-size floor (precision floor)
    max_depth=None,
    discrete_treatment=True,
    honest=True,                            # honest sample-splitting *within* trees
    cv=5,                                   # 5-fold cross-fitting for nuisance
    random_state=SEED,
)
forest.fit(Y=Y, T=T, X=X, W=W, groups=clusters)

# Per-unit CATE predictions WITH variance, on the held-out folds.
tau_hat, var_hat = forest.effect_inference(X).predict_mean(), forest.effect_inference(X).var
se_hat = np.sqrt(var_hat)
out = pd.DataFrame({"tau_hat": tau_hat, "se_hat": se_hat})
out.to_parquet("cate/cate-predictions.parquet")

Grading on the script:

  • Determinism (3 pts): SEED is passed into every randomness consumer (model_t, model_y, the forest itself).
  • Cluster-aware cross-fitting (3 pts): the groups=clusters argument is passed (or, if your library doesn’t support it natively, you implement custom cross-fitting that respects clusters; the README of econml describes the workaround).
  • honest=True (2 pts): the leaf-honesty flag is on; without it the BLP inference in Part 4 is biased.
  • Reasonable hyperparameters (2 pts): n_estimators >= 1000, min_samples_leaf calibrated to your sample size (rule of thumb: \min(50, N/100)). State your choices and their justification in the memo.

(b) Sanity-check the fit (10 pts). Three sanity checks, each a single line in the memo with a number:

  1. Overall ATE recovered. Compute forest.ate_inference(X).predict_mean() (the forest-implied ATE) and compare to your headline ATE from Chapter 4. They should agree to about \pm 1 standard error; a forest ATE that differs by more than 2 SE from the headline indicates either a bug or a substantive difference (the forest is using doubly-robust DML and your headline may not be — state the methodological gap explicitly).
  2. CATE-prediction sample variance. Compute \text{Var}(\hat\tau(\mathbf{x}_i)) across units. If the variance is essentially zero (less than the median \text{Var}(\hat\tau_i) from the noise alone), the forest is not finding heterogeneity — the data look like there is no CATE function at all. Report the variance and its ratio to the median per-unit variance.
  3. Leaf-size distribution. Plot the histogram of leaf sizes (or report the 10th and 90th percentiles). If most leaves have < 50 observations, the forest is fragmented and the CATE predictions are noisy; if most leaves have > N/10 observations, the forest is barely splitting and the CATE is nearly the ATE. Calibrate by adjusting min_samples_leaf if needed and re-running, but only before you run the BLP. Once you run the BLP, the forest is frozen.

Part 3 — The CATE histogram and the heterogeneity test (15 points)

(a) CATE histogram (5 pts). Build cate/cate-histogram.pdf showing the distribution of \hat\tau(\mathbf{x}_i) across units. The figure has:

  • The vertical line at the overall ATE (annotated, “ATE = X.XX pp”).
  • The shaded interval [ATE - 1.96 \cdot SE_{\text{ATE}}, ATE + 1.96 \cdot SE_{\text{ATE}}] (the 95% CI on the ATE, in grey).
  • The histogram of the per-unit \hat\tau_i (in blue, with a kernel-density overlay if your sample is small).
  • One-line annotation: “If the histogram is narrower than the ATE CI, there is no detectable heterogeneity.”

The visual rhetoric: a wide-relative-to-ATE-CI histogram shows the forest is detecting heterogeneity; a narrow histogram shows the forest is finding the ATE and not much more. This figure is the answer to the conference-question “is the effect heterogeneous?” before you do any formal inference.

(b) Heterogeneity F-test (5 pts). From the BLP regression in Part 4, the F-statistic for joint significance of the moderators is the standard test of “is there detectable heterogeneity along the moderator dimensions you pre-specified?” Report the F-statistic and the p-value. Under the null of constant treatment effect (no heterogeneity in the moderators), \hat\tau(\mathbf{x}_i) is constant in \mathbf{x}_i and all BLP coefficients are zero in expectation. A p < 0.05 is detectable heterogeneity jointly across moderators; a p > 0.10 suggests reporting just the overall ATE and noting that pre-specified heterogeneity was tested and not found.

(c) Interpret honestly (5 pts). Two cases, two paragraphs in the memo:

  • Case 1: heterogeneity detected. Write: “The pre-specified moderators jointly produce significant heterogeneity in the treatment effect (F = X.XX, p = 0.0YY). The CATE histogram (Figure C1) shows the per-unit treatment effects range from [10th percentile] to [90th percentile], a spread of [Y] pp around an ATE of [Z] pp.”
  • Case 2: no heterogeneity detected. Write: “The pre-specified moderators do not produce detectable heterogeneity (F = X.XX, p = 0.YY). The CATE histogram (Figure C1) is concentrated near the ATE [Z] pp, consistent with a constant treatment effect across the moderator space we pre-specified. We do not report subgroup ATTs as a primary result for this reason.”

Honesty in Case 2 is harder than in Case 1, because authors are reluctant to “find nothing.” But a forest that finds nothing on a pre-registered moderator list is a finding — it tells the reader that the heterogeneity stories the discussant suggested do not survive a flexible non-parametric test. That is publishable.

Part 4 — The honest best-linear-projection (25 points)

(a) The BLP regression (10 pts). The best-linear-projection (BLP) of the CATE function onto the moderators is the regression:

\hat\tau(\mathbf{x}_i) = \beta_0 + \sum_{j=1}^{p} \beta_j X_{ij} + u_i

estimated on the held-out predictions \hat\tau(\mathbf{x}_i) from the cross-fitted forest, with HC1 standard errors. The \beta_j coefficients are the linear projections of the (potentially non-linear) CATE function onto each moderator — they tell you whether, on average across the moderator space, an increase in X_j raises or lowers the treatment effect. This is the auditable summary of the forest’s heterogeneity finding.

# cate/forest.py continued — honest BLP from the cross-fitted predictions
import statsmodels.api as sm
df_blp = pd.read_parquet("cate/cate-predictions.parquet")
df_blp[X_cols] = X  # the moderators
X_blp = sm.add_constant(df_blp[X_cols])
y_blp = df_blp["tau_hat"]
blp = sm.OLS(y_blp, X_blp).fit(cov_type="HC1")
print(blp.summary())
with open("cate/blp.txt", "w") as fh:
    fh.write(str(blp.summary()))

Report each coefficient with its standard error and t-statistic in the memo as a four-column table. The pedagogical point is that the BLP summarizes the forest’s heterogeneity in the same units a reader is used to (regression coefficients) — a reader who does not know what a causal forest is can still read the BLP and understand the heterogeneity finding.

(b) Why the BLP is “honest” (5 pts). The honesty comes from three properties, each of which you confirmed in Parts 1-3:

  1. The moderators in the BLP were pre-registered (Part 1) — no fishing in the moderator space.
  2. The \hat\tau(\mathbf{x}_i) predictions came from cross-fitted forests (Part 2) — each unit’s prediction is from a forest that did not see that unit.
  3. Each tree split the training data into a honest leaf-structure-vs-leaf-mean split — the leaf means \hat\tau at each leaf are unbiased for the local CATE.

Write these three properties in the memo with one sentence each, citing Athey, Tibshirani, and Wager (2019) — “Generalized Random Forests,” Annals of Statistics 47(2):1148–1178. Without all three, the BLP standard errors are biased and the t-statistics lie.

(c) F-test for joint heterogeneity (5 pts). From the BLP output, the F-statistic for H_0: \beta_1 = \beta_2 = \dots = \beta_p = 0 (all moderator coefficients zero) is the formal test of whether the CATE varies along the pre-specified moderator directions. Report F, the degrees of freedom (p, N-p-1), and the p-value. Under the joint null of no heterogeneity, F is centered at 1; under the alternative, F grows with the strength of the moderation. A p < 0.05 is the result; a p < 0.001 is strong heterogeneity; a p > 0.10 is the “no heterogeneity detected” case.

(d) Per-moderator interpretation, in human units (5 pts). For each moderator X_j where \hat\beta_j is significant at the 5% level, write one sentence: “A one-[unit] increase in X_j is associated with a [coefficient]-pp change in the treatment effect, holding the other moderators at their mean (BLP, HC1 SE).” The “in human units” part is mandatory: a moderator measured in standard deviations should be reported in standard deviations; a moderator measured in dollars should be reported in dollars. Do not report unitless \beta_j’s in the paper — readers cannot calibrate them.

Part 5 — The CATE-vs-moderator plot (10 points)

(a) Pick the strongest moderator from the BLP (3 pts). Identify the moderator with the largest |t_j| in the BLP — the one driving the heterogeneity finding. Call this X^*. This is the moderator that goes on the x-axis of the second figure.

(b) Build cate/cate-vs-moderator.pdf (5 pts). A scatter plot with X^* on the x-axis and \hat\tau(\mathbf{x}_i) on the y-axis, with a LOESS or polynomial-fit overlay showing the trend, and a horizontal line at the overall ATE for reference. The figure looks like:

  • Scattered points (alpha = 0.3, small markers): the per-unit CATE predictions.
  • LOESS curve (degree-1, fraction = 0.3, in red): the smoothed CATE-as-function-of-X^*.
  • Horizontal grey line: the overall ATE.
  • Axis labels in human units (e.g., “Examination intensity (1=light, 2=moderate, 3=intense)” not “X_1”).
  • Annotation: “BLP coefficient on X^*: \hat\beta = [\text{value}] (t = [value]).”

The visual rhetoric: a clear monotone trend in the LOESS curve confirms the BLP’s directional finding visually; a flat LOESS despite a significant \hat\beta flags that the heterogeneity is being driven by a non-linear or interaction structure the BLP cannot capture and the figure cannot show — which is itself a publishable caveat.

(c) Optional second moderator (2 pts). If your BLP has two significant moderators, optionally produce a panel-of-two figure showing CATE-vs-X^* on the left and CATE-vs-X^{**} on the right. Skip if only one moderator is significant.

Part 6 — The paper paragraph (10 points)

The output that matters most is the paragraph that goes in §5 of your paper.

(a) Draft the paragraph (8 pts). Three to five sentences in memo.md, formatted as paper-ready prose (not bullet points, not a table):

  1. What you did. “We complement the per-subgroup ATT analysis of Section 4.3 with a causal-forest estimate of the conditional average treatment effect (CATE) following Athey, Tibshirani, and Wager (2019)…”
  2. What was pre-registered. “The moderator set (\mathbf{X}_i including [list]) was pre-specified in our pre-analysis plan committed at [date] before the forest was fit (commit hash X)…”
  3. What the forest found. “The CATE varies meaningfully along [moderator dimension], with \hat\beta_{X^*} = [\text{value}] in the best-linear-projection (HC1 t = [value]); the F-test for joint heterogeneity rejects constant treatment effects at the [level] level (F = [value], p = [value])…”
  4. What it means. “The substantive interpretation: [a one-sentence translation of the heterogeneity finding into the paper’s argument — e.g., ‘the policy works where the examination is intense and is essentially inert where it is not, consistent with the contract-form mechanism in Section 2’]…”
  5. What you cannot claim. “We do not claim the moderator coefficients in the BLP identify causal effects of the moderators themselves; the BLP is a projection of the heterogeneity onto a linear summary, not an identification statement about each moderator.”

The fifth sentence is the load-bearing one. It is the discipline that distinguishes a forest report from a regression report: the forest tells you where the effect lives, not why. The “why” question is PS 10.4’s mediation analysis.

(b) Connect to PS 10.2’s multiple-testing correction (2 pts). Add one sentence: “The BLP’s F-test is a single joint test and does not require multiple-testing adjustment across moderators (which would be needed only if we reported per-moderator effects as separate claims); the per-subgroup ATTs in Table 5 are independently corrected by Romano-Wolf step-down (PS 10.2 §3).” This sentence is what prevents a referee from claiming you double-corrected or under-corrected — it names which family is which.

The bridge to PS 10.4

PS 10.3 told you where the effect lives — which units (characterized by their moderators) get the most treatment effect. PS 10.4 will ask why: through what mediator does the treatment work, and is that mediator a clean mediator (an honest channel) or a bad control (a post-treatment variable that, if conditioned on, induces collider bias). The CATE-vs-moderator scatter from Part 5 is often the visual motivation for the mediation analysis — when you see that intense-examination counties get the largest treatment effect, the next question is what does an intense examination actually do, and the answer is a mediation model.

The paragraph in §5 that names the BLP F-test and the moderator coefficients is what turns “the ATE is X” into “the ATE varies along these pre-specified dimensions, and here is how” — a credibility upgrade that is the standard at JFE-tier journals.

5. `cate/cate-histogram.pdf` and `cate/cate-vs-moderator.pdf` — the two figures (Part 5), production-grade per Ch 9.3 §2 (matplotlib, 6×4 inches, deterministic).
6. `cate/memo.md` — the one-page memo (Part 6) that interprets the BLP, names the surviving heterogeneity direction(s), and writes the paragraph that goes in §5 of the paper.

## Part 1 — The pre-analysis-plan extract (20 points)

The single sin this part prevents: **letting the data tell you which moderators to test, then dressing the answer up as if you had pre-specified them.** A causal forest's value comes from the discipline imposed *before* it sees the data, not from the forest itself.

**(a) Name the moderator list (8 pts).** Write, in `pap-extract.md`, the bulleted list of moderators $\mathbf{X}_i$ — the covariates the forest is allowed to use to predict $\tau_i$. The list should have **between 3 and 12 moderators**. Fewer than 3 and the forest cannot find heterogeneity; more than 12 and the forest will overfit and the BLP will lose power (recall the multiple-testing problem of Ch 10.2 returns at every leaf split). Each moderator gets a one-sentence justification of the form: *"$X_j$ is included because [theoretical / institutional reason connecting $X_j$ to the mechanism]."* For Maya, the four moderators are intensity (FFIEC exam scope), county minority share (decennial census), MSA indicator (OMB definition), CRA-evaluation-period overlap (FFIEC CRA calendar). For Leah, the six moderators are industry FE (SIC 2-digit), market cap (CRSP), R&D intensity (Compustat XRD/AT), patent-stock age (USPTO), citation count (USPTO + Bessen), and litigation history (Stanford NPE database). State *your* six.

**(b) Outcome, treatment, sample (3 pts).** Restate the outcome variable, the treatment variable, the sample (same as your headline regression — *do not* expand the sample for CATE, that is a fork), the time window, and the unit of analysis (county-year, firm-year, etc.). Three sentences total. This is the spec the forest will be fit on; locking it here prevents drift.

**(c) Cross-validation / sample-splitting protocol (5 pts).** State the *honesty* protocol you will use. The default this textbook recommends, following Athey-Wager (2019):

- **5-fold cross-fitting**: each unit's $\hat\tau(\mathbf{x}_i)$ is estimated from a forest trained on a fold that does *not* contain that unit. This is the property that makes the BLP standard errors valid.
- **Within each fold, honest sample-splitting at the leaf level**: each tree splits the training data into two random halves; one half determines tree structure (the splits), the other half estimates the leaf-mean $\hat\tau$ at each leaf. This is the property that makes the per-unit $\hat\tau(\mathbf{x}_i)$ asymptotically normal.
- **Clustering at the cluster level your headline regression used**: cross-fitting folds are formed by *whole clusters*, not units, so the same cluster never appears in both train and test. For panel data, this is usually whole *firms* or whole *counties* (not firm-years).

Write this protocol as three sentences. It is the part the forest's `subforest_size` and `honest=True` arguments in `econml` (or `honesty=TRUE` in `grf`) implement; the protocol-as-prose is what your PAP-extract documents.

**(d) Lock the PAP-extract and tag the commit (4 pts).** Commit `cate/pap-extract.md` to the repository with the message `cate: PAP-extract locked, moderators pre-specified`, then tag the commit `cate-pap-locked`. From this commit forward, the moderator list is frozen — adding a moderator after the forest has produced an interesting CATE-vs-X plot is fishing, and is detectable by `git log -- cate/pap-extract.md`. If you must add a moderator (because the forest exposed a feature you had not considered), do it transparently: a new commit, a new tag (`cate-pap-revised`), and a paragraph in the memo (Part 6) explaining why. *The transparency makes the revision honest; the silent edit is the dishonesty.*

## Part 2 — Fit the causal forest with honest sample-splitting (20 points)

**(a) Implement with `econml.dml.CausalForestDML` (10 pts).** The script `cate/forest.py` looks like this skeleton:

```python
# cate/forest.py — causal forest with honest sample-splitting and cross-fitting
import numpy as np
import pandas as pd
from econml.dml import CausalForestDML
from sklearn.linear_model import LassoCV
from sklearn.ensemble import RandomForestRegressor, RandomForestClassifier
from config import SEED

rng = np.random.default_rng(SEED)
df = pd.read_parquet("data/processed/analysis.parquet")

# Moderators X (the heterogeneity-driving features), treatment T, outcome Y,
# nuisance controls W (all the controls your headline regression had).
X_cols = ["intensity_bin", "minority_share", "msa", "cra_overlap"]
W_cols = ["unemp", "median_income", "share_white", "median_age"]  # confounders
T_col, Y_col = "treat_post", "denial_gap"
cluster_col = "county_fips"

X = df[X_cols].to_numpy()
W = df[W_cols].to_numpy()
T = df[T_col].to_numpy()
Y = df[Y_col].to_numpy()
clusters = df[cluster_col].to_numpy()

forest = CausalForestDML(
    model_t=RandomForestClassifier(n_estimators=200, random_state=SEED, n_jobs=-1),
    model_y=RandomForestRegressor(n_estimators=200, random_state=SEED, n_jobs=-1),
    n_estimators=2000,                      # trees in the causal forest itself
    min_samples_leaf=50,                    # leaf-size floor (precision floor)
    max_depth=None,
    discrete_treatment=True,
    honest=True,                            # honest sample-splitting *within* trees
    cv=5,                                   # 5-fold cross-fitting for nuisance
    random_state=SEED,
)
forest.fit(Y=Y, T=T, X=X, W=W, groups=clusters)

# Per-unit CATE predictions WITH variance, on the held-out folds.
tau_hat, var_hat = forest.effect_inference(X).predict_mean(), forest.effect_inference(X).var
se_hat = np.sqrt(var_hat)
out = pd.DataFrame({"tau_hat": tau_hat, "se_hat": se_hat})
out.to_parquet("cate/cate-predictions.parquet")
```

Grading on the script:

- **Determinism (3 pts):** `SEED` is passed into every randomness consumer (`model_t`, `model_y`, the forest itself).
- **Cluster-aware cross-fitting (3 pts):** the `groups=clusters` argument is passed (or, if your library doesn't support it natively, you implement custom cross-fitting that respects clusters; the README of `econml` describes the workaround).
- **`honest=True` (2 pts):** the leaf-honesty flag is on; without it the BLP inference in Part 4 is biased.
- **Reasonable hyperparameters (2 pts):** `n_estimators >= 1000`, `min_samples_leaf` calibrated to your sample size (rule of thumb: $\min(50, N/100)$). State your choices and their justification in the memo.

**(b) Sanity-check the fit (10 pts).** Three sanity checks, each a single line in the memo with a number:

1. *Overall ATE recovered.* Compute `forest.ate_inference(X).predict_mean()` (the forest-implied ATE) and compare to your headline ATE from Chapter 4. They should agree to about $\pm 1$ standard error; a forest ATE that differs by more than 2 SE from the headline indicates either a bug or a substantive difference (the forest is using doubly-robust DML and your headline may not be — state the methodological gap explicitly).
2. *CATE-prediction sample variance.* Compute $\text{Var}(\hat\tau(\mathbf{x}_i))$ across units. If the variance is essentially zero (less than the median $\text{Var}(\hat\tau_i)$ from the noise alone), the forest is *not finding heterogeneity* — the data look like there is no CATE function at all. Report the variance and its ratio to the median per-unit variance.
3. *Leaf-size distribution.* Plot the histogram of leaf sizes (or report the 10th and 90th percentiles). If most leaves have $< 50$ observations, the forest is fragmented and the CATE predictions are noisy; if most leaves have $> N/10$ observations, the forest is barely splitting and the CATE is nearly the ATE. Calibrate by adjusting `min_samples_leaf` if needed and re-running, *but only before you run the BLP*. Once you run the BLP, the forest is frozen.

## Part 3 — The CATE histogram and the heterogeneity test (15 points)

**(a) CATE histogram (5 pts).** Build `cate/cate-histogram.pdf` showing the distribution of $\hat\tau(\mathbf{x}_i)$ across units. The figure has:

- The vertical line at the overall ATE (annotated, "ATE = X.XX pp").
- The shaded interval `[ATE - 1.96 \cdot SE_{\text{ATE}}, ATE + 1.96 \cdot SE_{\text{ATE}}]` (the 95% CI on the ATE, in grey).
- The histogram of the per-unit $\hat\tau_i$ (in blue, with a kernel-density overlay if your sample is small).
- One-line annotation: "If the histogram is narrower than the ATE CI, there is no detectable heterogeneity."

The visual rhetoric: a wide-relative-to-ATE-CI histogram shows the forest *is* detecting heterogeneity; a narrow histogram shows the forest is finding the ATE and not much more. This figure is the answer to the conference-question "is the effect heterogeneous?" *before* you do any formal inference.

**(b) Heterogeneity F-test (5 pts).** From the BLP regression in Part 4, the F-statistic for joint significance of the moderators is the standard test of "is there detectable heterogeneity along the moderator dimensions you pre-specified?" Report the F-statistic and the p-value. Under the null of constant treatment effect (no heterogeneity in the moderators), $\hat\tau(\mathbf{x}_i)$ is constant in $\mathbf{x}_i$ and all BLP coefficients are zero in expectation. A p < 0.05 is detectable heterogeneity *jointly across moderators*; a p > 0.10 suggests reporting just the overall ATE and noting that pre-specified heterogeneity was tested and not found.

**(c) Interpret honestly (5 pts).** Two cases, two paragraphs in the memo:

- *Case 1: heterogeneity detected.* Write: *"The pre-specified moderators jointly produce significant heterogeneity in the treatment effect (F = X.XX, p = 0.0YY). The CATE histogram (Figure C1) shows the per-unit treatment effects range from [10th percentile] to [90th percentile], a spread of [Y] pp around an ATE of [Z] pp."*
- *Case 2: no heterogeneity detected.* Write: *"The pre-specified moderators do not produce detectable heterogeneity (F = X.XX, p = 0.YY). The CATE histogram (Figure C1) is concentrated near the ATE [Z] pp, consistent with a constant treatment effect across the moderator space we pre-specified. We do not report subgroup ATTs as a primary result for this reason."*

Honesty in Case 2 is harder than in Case 1, because authors are reluctant to "find nothing." But a forest that finds nothing on a pre-registered moderator list is a *finding* — it tells the reader that the heterogeneity stories the discussant suggested do not survive a flexible non-parametric test. That is publishable.

## Part 4 — The honest best-linear-projection (25 points)

**(a) The BLP regression (10 pts).** The best-linear-projection (BLP) of the CATE function onto the moderators is the regression:

$$ \hat\tau(\mathbf{x}_i) = \beta_0 + \sum_{j=1}^{p} \beta_j X_{ij} + u_i $$

estimated on the held-out predictions $\hat\tau(\mathbf{x}_i)$ from the cross-fitted forest, with HC1 standard errors. The $\beta_j$ coefficients are the *linear projections* of the (potentially non-linear) CATE function onto each moderator — they tell you whether, *on average across the moderator space*, an increase in $X_j$ raises or lowers the treatment effect. This is the auditable summary of the forest's heterogeneity finding.

```python
# cate/forest.py continued — honest BLP from the cross-fitted predictions
import statsmodels.api as sm
df_blp = pd.read_parquet("cate/cate-predictions.parquet")
df_blp[X_cols] = X  # the moderators
X_blp = sm.add_constant(df_blp[X_cols])
y_blp = df_blp["tau_hat"]
blp = sm.OLS(y_blp, X_blp).fit(cov_type="HC1")
print(blp.summary())
with open("cate/blp.txt", "w") as fh:
    fh.write(str(blp.summary()))
```

Report each coefficient with its standard error and t-statistic in the memo as a four-column table. The pedagogical point is that the BLP *summarizes* the forest's heterogeneity in the same units a reader is used to (regression coefficients) — a reader who does not know what a causal forest is can still read the BLP and understand the heterogeneity finding.

**(b) Why the BLP is "honest" (5 pts).** The honesty comes from three properties, each of which you confirmed in Parts 1-3:

1. The moderators in the BLP were *pre-registered* (Part 1) — no fishing in the moderator space.
2. The $\hat\tau(\mathbf{x}_i)$ predictions came from *cross-fitted* forests (Part 2) — each unit's prediction is from a forest that did not see that unit.
3. Each tree split the training data into a *honest* leaf-structure-vs-leaf-mean split — the leaf means $\hat\tau$ at each leaf are unbiased for the local CATE.

Write these three properties in the memo with one sentence each, citing Athey, Tibshirani, and Wager (2019) — "Generalized Random Forests," *Annals of Statistics* 47(2):1148–1178. Without all three, the BLP standard errors are biased and the t-statistics lie.

**(c) F-test for joint heterogeneity (5 pts).** From the BLP output, the F-statistic for $H_0: \beta_1 = \beta_2 = \dots = \beta_p = 0$ (all moderator coefficients zero) is the formal test of whether the CATE varies along the pre-specified moderator directions. Report F, the degrees of freedom $(p, N-p-1)$, and the p-value. Under the joint null of no heterogeneity, F is centered at 1; under the alternative, F grows with the strength of the moderation. A p < 0.05 is the result; a p < 0.001 is strong heterogeneity; a p > 0.10 is the "no heterogeneity detected" case.

**(d) Per-moderator interpretation, in human units (5 pts).** For each moderator $X_j$ where $\hat\beta_j$ is significant at the 5% level, write one sentence: *"A one-[unit] increase in $X_j$ is associated with a [coefficient]-pp change in the treatment effect, holding the other moderators at their mean (BLP, HC1 SE)."* The "in human units" part is mandatory: a moderator measured in standard deviations should be reported in standard deviations; a moderator measured in dollars should be reported in dollars. Do not report unitless $\beta_j$'s in the paper — readers cannot calibrate them.

## Part 5 — The CATE-vs-moderator plot (10 points)

**(a) Pick the strongest moderator from the BLP (3 pts).** Identify the moderator with the largest $|t_j|$ in the BLP — the one driving the heterogeneity finding. Call this $X^*$. This is the moderator that goes on the x-axis of the second figure.

**(b) Build `cate/cate-vs-moderator.pdf` (5 pts).** A scatter plot with $X^*$ on the x-axis and $\hat\tau(\mathbf{x}_i)$ on the y-axis, with a LOESS or polynomial-fit overlay showing the trend, and a horizontal line at the overall ATE for reference. The figure looks like:

- Scattered points (alpha = 0.3, small markers): the per-unit CATE predictions.
- LOESS curve (degree-1, fraction = 0.3, in red): the smoothed CATE-as-function-of-$X^*$.
- Horizontal grey line: the overall ATE.
- Axis labels in human units (e.g., "Examination intensity (1=light, 2=moderate, 3=intense)" not "X_1").
- Annotation: *"BLP coefficient on $X^*$: $\hat\beta = [\text{value}]$ (t = [value])."*

The visual rhetoric: a clear monotone trend in the LOESS curve confirms the BLP's directional finding visually; a flat LOESS despite a significant $\hat\beta$ flags that the heterogeneity is being driven by a non-linear or interaction structure the BLP cannot capture and the figure cannot show — which is itself a publishable caveat.

**(c) Optional second moderator (2 pts).** If your BLP has two significant moderators, optionally produce a panel-of-two figure showing CATE-vs-$X^*$ on the left and CATE-vs-$X^{**}$ on the right. Skip if only one moderator is significant.

## Part 6 — The paper paragraph (10 points)

The output that matters most is the paragraph that goes in §5 of your paper.

**(a) Draft the paragraph (8 pts).** Three to five sentences in `memo.md`, formatted as paper-ready prose (not bullet points, not a table):

1. *What you did.* "We complement the per-subgroup ATT analysis of Section 4.3 with a causal-forest estimate of the conditional average treatment effect (CATE) following Athey, Tibshirani, and Wager (2019)..."
2. *What was pre-registered.* "The moderator set ($\mathbf{X}_i$ including [list]) was pre-specified in our pre-analysis plan committed at [date] before the forest was fit (commit hash $X$)..."
3. *What the forest found.* "The CATE varies meaningfully along [moderator dimension], with $\hat\beta_{X^*} = [\text{value}]$ in the best-linear-projection (HC1 t = [value]); the F-test for joint heterogeneity rejects constant treatment effects at the [level] level (F = [value], p = [value])..."
4. *What it means.* "The substantive interpretation: [a one-sentence translation of the heterogeneity finding into the paper's argument — e.g., 'the policy works where the examination is intense and is essentially inert where it is not, consistent with the contract-form mechanism in Section 2']..."
5. *What you cannot claim.* "We do not claim the moderator coefficients in the BLP identify causal effects of the moderators themselves; the BLP is a projection of the heterogeneity onto a linear summary, not an identification statement about each moderator."

The fifth sentence is the load-bearing one. It is the discipline that distinguishes a forest report from a regression report: the forest tells you *where the effect lives*, not *why*. The "why" question is PS 10.4's mediation analysis.

**(b) Connect to PS 10.2's multiple-testing correction (2 pts).** Add one sentence: *"The BLP's F-test is a single joint test and does not require multiple-testing adjustment across moderators (which would be needed only if we reported per-moderator effects as separate claims); the per-subgroup ATTs in Table 5 are independently corrected by Romano-Wolf step-down (PS 10.2 §3)."* This sentence is what prevents a referee from claiming you double-corrected or under-corrected — it names which family is which.

## The bridge to PS 10.4

PS 10.3 told you *where* the effect lives — which units (characterized by their moderators) get the most treatment effect. PS 10.4 will ask *why*: through what mediator does the treatment work, and is that mediator a clean mediator (an honest channel) or a bad control (a post-treatment variable that, if conditioned on, induces collider bias). The CATE-vs-moderator scatter from Part 5 is often the visual motivation for the mediation analysis — when you see that intense-examination counties get the largest treatment effect, the next question is *what does an intense examination actually do*, and the answer is a mediation model.
