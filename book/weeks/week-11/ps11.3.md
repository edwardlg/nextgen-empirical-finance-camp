A second boundary. **The twelve variants are not twelve different *plots* — they are twelve different *renderings of the same plot*.** The underlying coefficient estimates, the same error bars, the same x-axis values. What varies is the typography. You will start to feel, after the eighth variant, that nine of them are *worse* than your first instinct — that is the lesson. The remaining three or four are genuine alternatives between which the choice is a judgment call. The judgment call is what the memo defends.

## Part 1 — Estimate (or load) the coefficients (15 points)

The single sin this part prevents: **re-running estimation on every plot iteration.** Separate the estimation step from the plotting step.

**(a) Decide what your figure shows (5 pts).** The chapter taught three figure types. Pick the one your headline finding most needs:

- *Event-study plot* — for DiD or staggered DiD designs. *x*-axis: event time $\tau$. *y*-axis: $\hat{\delta}_\tau$ with confidence band. Maya, Devon (event window around the May 2022 de-peg), Priya (event window around the flood-map revision).
- *Specification-curve plot* — for papers with a large spec menu. *x*-axis: specification index (sorted by coefficient). *y*-axis: coefficient with band. Useful for *all* projects as a robustness figure, but in the paper's body only when the spec robustness *is* the headline contribution.
- *Heterogeneity forest* — for papers with pre-specified subgroups. *y*-axis (rows): subgroup labels. *x*-axis: coefficient with band. Sam (by year-of-trading), Leah (by IPC technology class).

Pick *one* type, state the choice in `selection-memo.md`'s first sentence. PS 11.3 produces twelve variants of that *one* figure; the other two figure types appear elsewhere in the paper.

**(b) Estimate the coefficients with cluster-robust SEs (5 pts).** Re-use the regression objects from PS 11.2 if applicable. For the event-study case, estimate the leads and lags as separate coefficients (e.g., $\tau \in \{-3, -2, -1, 0, +1, +2, +3\}$, with $\tau = -1$ as the omitted baseline category). For the specification-curve case, run the full spec menu and store the coefficient and 95% CI bound for each. For the heterogeneity case, estimate the coefficient × subgroup interaction (or run the regression separately on each subgroup) and store the coefficients and CIs.

```python
# make_figure.py — Maya's event-study coefficients (verified)
import numpy as np
import pandas as pd
import pyfixest as pf

rng = np.random.default_rng(42)
N_county, T = 1000, 7
years = np.arange(2014, 2021)
treated_idx = rng.choice(N_county, 614, replace=False)
panel = pd.DataFrame({
    "county": np.repeat(np.arange(N_county), T),
    "year": np.tile(years, N_county),
})
panel["treated"] = panel["county"].isin(treated_idx).astype(int)
panel["event_time"] = panel["year"] - 2017
panel["denial_gap"] = (
    6.1
    - np.where(
        (panel["treated"] == 1) & (panel["event_time"] >= 0),
        1.4 * (1 + 0.25 * panel["event_time"]),
        0,
    )
    + 0.2 * (panel["year"] - 2017)
    + rng.normal(0, 1.5, len(panel))
)
# Event-study: separate dummy for each event time, omit tau = -1
for tau in [-3, -2, 0, 1, 2, 3]:
    panel[f"D_{tau}"] = (
        ((panel["treated"] == 1) & (panel["event_time"] == tau))
    ).astype(int)
es = pf.feols(
    "denial_gap ~ D_-3 + D_-2 + D_0 + D_1 + D_2 + D_3 | county + year",
    data=panel, vcov={"CRV1": "county"},
)
print(es.coef())
print(es.se())
```

Save the resulting `coefs` and `ses` Pandas Series to disk (e.g., `event_study_coefs.csv`) so the plotting variants in Part 2 read from the same coefficient inputs and you do not re-estimate every time you tweak the typography.

**(c) Verify the coefficient signs and rough magnitudes (5 pts).** Print the coefficient table. For Maya's event study, the *pre-period* coefficients $D_{-3}, D_{-2}$ should be near zero (the parallel-trends assumption); the omitted category $D_{-1} \equiv 0$ is the reference; the *post-period* coefficients $D_0, D_1, D_2, D_3$ should be negative with magnitudes increasing over event time (the treatment-effect dynamics). If a pre-period coefficient is statistically distinguishable from zero, you have a parallel-trends problem and the figure will surface it — this is one of the things the figure is *for*.

## Part 2 — Produce the twelve variants (40 points)

The twelve variants are organized into four groups of three. Each group varies one design dimension while holding the others constant. Run each variant. Save as a PNG. Build the contact sheet as a 4×3 grid.

**(a) Variants 1–3: error band style (10 pts).**

- *Variant 1: Vertical error bars at each point.* `plt.errorbar(x, coefs, yerr=1.96 * ses, fmt='o', ecolor='gray', capsize=3)`. The classic. Easy to read for sparse $x$-axes (Maya's 7 event-time points). Cluttered for dense $x$-axes.
- *Variant 2: Shaded confidence band as polygon.* `plt.fill_between(x, coefs - 1.96 * ses, coefs + 1.96 * ses, alpha=0.2)`, with `plt.plot(x, coefs)` on top. The data-ink-maximal choice. Used in modern top-journal event studies (Callaway and Sant'Anna 2021's figures).
- *Variant 3: Whiskers without caps + line connecting points.* `plt.errorbar(..., capsize=0)` plus a connecting line. A middle ground. Some readers find the connecting line implies continuity (it does not — the event-time dummies are discrete) and prefer to omit it.

**(b) Variants 4–6: color and demarcation (10 pts).**

- *Variant 4: Single accent color (#1f77b4 default blue) for both points and band.* The Tufte default.
- *Variant 5: Two colors — one for pre, one for post.* Pre-treatment in gray (`#888888`), post-treatment in accent. The visual emphasizes that pre is the test, post is the result.
- *Variant 6: Color by significance.* Points colored darker where $|t| > 1.96$, lighter where $|t| < 1.96$. Some readers find this distracting; some find it efficient.

**(c) Variants 7–9: axis decoration (10 pts).**

- *Variant 7: No grid, minimal axis decoration.* Tufte-maximal. The two reference lines (vertical at $\tau = 0$, horizontal at $y = 0$) are the only non-data lines.
- *Variant 8: Faint horizontal grid only.* `plt.grid(axis='y', alpha=0.2)`. Some readers like the grid for reading values; others find it chartjunk.
- *Variant 9: Box around the plot area + grid.* The `matplotlib` default. Show this to demonstrate why you reject it: too much chartjunk.

**(d) Variants 10–12: x-axis labeling (10 pts).**

- *Variant 10: Event-time integers $-3, -2, -1, 0, +1, +2, +3$.* The canonical.
- *Variant 11: Calendar years 2014, 2015, …, 2020.* Wrong for a staggered design (event time is the right axis), but show what calendar-year labels look like to demonstrate the contrast.
- *Variant 12: Event-time integers plus a calendar-year annotation under $\tau = 0$ ("2017 reform").* The dual-axis style — gives the reader both the event-time interpretation and the calendar anchor.

For each variant, save a 6×4 inch PNG at 150 DPI (high enough for the contact sheet to be legible, not so high that the file size is huge). Use a consistent random seed and the same coefficients across variants.

## Part 3 — Build the contact sheet (10 points)

The contact sheet is the artifact that lets you (and your advisor) see all twelve variants at once.

**(a) The 4×3 grid (5 pts).** Use `matplotlib.gridspec` or a manual `plt.subplots(4, 3, figsize=(12, 16))`. Each cell shows one variant at thumbnail size. Title each cell with the variant number ("V1: Error bars" through "V12: Event-time + calendar"). Save as `contact-sheet.png` at 300 DPI for the appendix and at 100 DPI for the inline view in `selection-memo.md`.

**(b) Adjacent grouping (3 pts).** Group the four families on adjacent rows: row 1 = error-band style (V1, V2, V3); row 2 = color (V4, V5, V6); row 3 = axis decoration (V7, V8, V9); row 4 = x-axis labeling (V10, V11, V12). This makes the per-row comparisons visually meaningful.

**(c) Annotated rejection (2 pts).** Cross out the rejected variants with a red diagonal line drawn over the thumbnail (use `plt.axline` or a `patches.Rectangle`). The rejected variants stand out at a glance; the chosen variant is uncrossed. The advisor reading the memo sees which one you picked without reading the prose.

## Part 4 — The selection memo (20 points)

The memo is the load-bearing artifact of this PS. It is what demonstrates that you can articulate the choice, not just make it.

**(a) Name your choice (3 pts).** First sentence: "We chose variant V$X$ for the manuscript because…" with one of: V2 (shaded band, no grid, single accent), V7 (no grid, minimal decoration), or some hybrid you assembled. Most students who run the exercise honestly choose V2 or V7 or a hybrid of the two. State the choice in the first sentence.

**(b) Why this and not the closest alternative (8 pts).** Write three sentences (~120 words) defending the choice against the *closest alternative*. The closest alternative is not the worst variant (V9 or V11); it is the variant you actually considered seriously. For most students this is V1 vs. V2 (error bars vs. shaded band) or V7 vs. V8 (no grid vs. faint grid). Defend the choice with reasons grounded in the data-ink discipline:

- *Data-ink ratio.* The chosen variant has higher data-ink ratio because [the band is data; the error bars also data but with caps that are decoration].
- *One-glance test.* The chosen variant lets the reader see the parallel-trends test (pre-period bands cross zero) in two seconds; the alternative buries it in a sea of marks.
- *Convention in your subfield.* Three recent papers in your target journal use [shaded bands / error bars / etc.]. Conformity to subfield norm has value when the figure is not the contribution.

**(c) Why each of the eleven others is wrong (6 pts).** Write one sentence per rejected variant. Eleven sentences. Each should name the *specific* failure: "V9: the box around the plot area adds 16 line-segments of ink that carry no data; rejected on data-ink grounds." "V11: calendar-year x-axis is incorrect for a staggered design with multiple cohorts; rejected for design-fidelity." Short, brutal, honest. The discipline of writing these out is what trains your eye.

**(d) The one-glance question (3 pts).** Have a peer who has not seen the figure look at it for 15 seconds. Ask them the four questions from Chapter 11.3 §1 (what is the x-axis, the y-axis, the comparison, the bottom-line claim?). Record their answers in the memo. If they got all four, the figure passes; if not, note the failed question and what the revision would be.

## Part 5 — Render the camera-ready (15 points)

**(a) The 6×4-inch PNG (5 pts).** Final figure. 300 DPI. PNG and PDF. Embedded fonts (`matplotlib.rcParams['pdf.fonttype'] = 42` in the script).

**(b) The caption (5 pts).** Write the figure caption that goes in the manuscript. The caption is *informative* — it should let the reader understand the figure without reading the body text. Three sentences:

1. *What the figure shows.* "Event-study estimates of the effect of the 2017 OCC fair-lending examination intensification on the county-level Black-White denial-rate gap."
2. *The axes and band.* "Event time $\tau = 0$ corresponds to the calendar year of treatment for each county; coefficients on event-time dummies are reported relative to $\tau = -1$. Shaded band is the 95% confidence interval computed from cluster-robust standard errors (clustered at the county level)."
3. *The bottom line.* "Pre-treatment coefficients are statistically indistinguishable from zero, supporting parallel trends; post-treatment coefficients are negative and grow in magnitude over event time, with the $\tau = +3$ coefficient of $-1.8$ pp implying a cumulative effect of roughly one-quarter of the pre-treatment gap."

The caption is its own three-sentence skeleton: *what / how / so-what*. Write all three.

**(c) The label and reference (5 pts).** Label the figure `\label{fig:event-study}` and reference it from the text where the parallel-trends discussion happens. Verify the reference compiles (no `??` in the PDF output). Three places the figure must be referenced from: (i) the introduction's findings paragraph (PS 11.4, paragraph 4), (ii) the design section, (iii) the results section. Confirm all three in the memo.

If not, the figure has not yet earned its place in the body.

## A note on figure types other than event studies

If your paper does not have a DiD design, the twelve-variant exercise still applies but with different design dimensions. For a **specification curve**: vary the sort order of the spec rows (by coefficient, by t-statistic, by p-value), the band style (CI vs. p-value-as-color), and the annotation density (label every fifth spec vs. label none). For a **heterogeneity forest**: vary the subgroup ordering (by point estimate, by subgroup size, by alphabetical label), the band style (CI vs. SE-as-whiskers), and the reference-zero line presentation. The four-family structure adapts; the discipline does not. Pick the type that best matches your design and run the exercise on it.
