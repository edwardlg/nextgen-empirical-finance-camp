# Lab 11 — Manuscript Build Lab: One Command from Raw DGP to Camera-Ready PDF

Week 8 produced a robustness battery that ran. Week 10 produced an appendix-grade extension of that battery and a six-page memo defending it. Week 11's chapters then turned the artifacts into manuscript sections: a 245-word abstract (Ch 11.1), publication-grade tables generated by `pyfixest`/`etable` (Ch 11.2), figures with consistent typography (Ch 11.3), and a five-paragraph introduction (Ch 11.4). What you do not yet have is a *single command that builds the entire PDF from the raw synthetic-DGP inputs*. That is what Lab 11 builds.

Here is the move this lab turns on. **A camera-ready paper is not a Word document with figures pasted in; it is a *derivation* — a sequence of deterministic steps that takes raw bytes and produces a PDF, with every number, table, figure, and citation traceable to a code path that regenerated it.** The day a referee asks "where did the 3.1pp number in your abstract come from?", the answer should be a file path and a line number, not a stale spreadsheet on someone's laptop. The single `make paper` target is the contract.

Nothing in Lab 11 requires the internet at run time. The raw inputs are produced by the synthetic DGP you ran in Week 7, frozen as Parquet on local disk. The LaTeX toolchain is `pdflatex` + `bibtex`, both shipped in TeXLive. The total wall-clock for a clean rebuild on a recent laptop is 90–180 seconds. Budget 4 hours for the lab itself.

---

## Learning goals

By the end of this lab you will be able to:

1. **Stand up the AEA LaTeX template** (`aea.cls`, the AEA bibliography style, and the `aea` reference-formatting macros) inside your capstone repository so the manuscript compiles to the same column widths, font sizes, and citation style as a *Journal of Financial and Quantitative Analysis* or *American Economic Review* submission.
2. **Generate every table from `pyfixest`/`etable`** with a single Python module that reads the cleaned panel, runs the headline and appendix regressions, and writes `.tex` fragments the manuscript `\input{}`s. No `\begin{tabular}` blocks typed by hand.
3. **Generate every figure from `matplotlib`** with the typography settings (font family, sizes, color palette) shared across figures via a `figstyle.py` module, so the figures look like they came from the same paper rather than five different students' Jupyter notebooks.
4. **Validate the BibTeX bibliography against the Week-7 verified-bibs list**, with a script that fails the build if any citation key in the manuscript is missing from the verified list or carries a `[CHECK]` tag.
5. **Wire the LaTeX compile into the `Makefile`** so `make paper` rebuilds analysis → tables → figures → bibliography → PDF in one command, and `make clean && make paper` reproduces the PDF byte-for-byte on a fresh clone.
6. **Stamp the PDF with a build manifest** — the git commit hash, the seed, the timestamp, and the SHA-256 of the cleaned panel — so a future reader (including future-you) can pin which version of the code produced any given PDF.

## Part 1 — The AEA template, sized and styled

The American Economic Association distributes a LaTeX class file (`aea.cls`) that sets the column widths, margins, font sizes, and bibliography style of all AEA journals. Three of Prof. Gao's papers — *Gao, He & Wu (2023, JFQA)*, *Elnahas, Gao, Hossain & Kim (2024, JFQA)*, and *Gao, Han, Kim & Pan (2024, JCF)* — were submitted via templates close to this format. JFQA is not an AEA journal, but its column structure is similar enough that the AEA template is a defensible camera-ready format for capstone purposes.

### 1.1 Acquire the template files

The AEA template is distributed by the AEA at `aeaweb.org/journals/policies/templates` and includes `aea.cls`, `aea.bst` (the bibliography style), and a `sample.tex` file demonstrating use. For the camp's offline-first design, the template files are vendored into the camp repository under `book/templates/aea/`. Copy them into your manuscript folder:

```bash
cp -r /path/to/camp-repo/book/templates/aea/* manuscript/aea/
ls manuscript/aea/
# Expected: aea.cls aea.bst aea-template.tex README-aea.md
```

If you cannot find the camp's vendored copy, download the template from the AEA's site once (a one-time fetch), commit the files to your repo, and never touch them again. Vendoring the template is the correct move: a paper that fails to compile in 2030 because the AEA changed their `.cls` file is not reproducible.

Notice three discipline moves. **First**, every section is `\input`ed from its own file, so the main file stays under 80 lines and each section is editable in isolation. **Second**, the author's affiliation footnote `\input`s a build stamp the `make paper` target regenerates every build, so the PDF always carries the commit hash. **Third**, the `\date{}` is `\input`ed from a generated file, so each rebuild stamps today's date without manual edits.

Devon's crypto-event-study capstone uses the same skeleton with renamed sections (`chain_data.tex`, `event_specification.tex`); Priya's climate-and-insurance capstone has an extra `taxonomy.tex` between `data.tex` and `design.tex`. The skeleton is the constant; the section content is what varies.

## Part 2 — Tables generated by `pyfixest`/`etable`

The Chapter 11.2 discipline was: every table is generated, never typed. Lab 11 operationalizes that discipline as a single `src/build/tables.py` module that reads the cleaned panel, runs every regression the paper reports, and writes `.tex` fragments the manuscript `\input`s. The `pyfixest` package's `etable()` function is the workhorse: it takes a list of fitted regression objects and emits a publication-grade LaTeX table with stars, standard errors, fixed-effects rows, and observation counts.

### 2.1 The `src/build/tables.py` module

Here is the structure for Maya's paper, generalizable to anyone's:

```python
"""src/build/tables.py — emit every manuscript table from the cleaned panel."""
from pathlib import Path
import argparse
import numpy as np
import pandas as pd
import pyfixest as pf

SEED = 42

def fit_table2(panel: pd.DataFrame) -> dict:
    """Headline DiD table: 4 columns of progressively richer specifications."""
    fits = {
        "(1) Bivariate": pf.feols("denial_gap ~ post_exam", data=panel, vcov={"CRV1": "county"}),
        "(2) + Year FE": pf.feols("denial_gap ~ post_exam | year", data=panel, vcov={"CRV1": "county"}),
        "(3) + County FE": pf.feols("denial_gap ~ post_exam | year + county", data=panel, vcov={"CRV1": "county"}),
        "(4) + Covariates": pf.feols(
            "denial_gap ~ post_exam + log_pop + median_income + unemp_rate | year + county",
            data=panel, vcov={"CRV1": "county"}),
    }
    return fits

def fit_table3(panel: pd.DataFrame) -> dict:
    """Heterogeneity: examination-history quartiles."""
    fits = {}
    for q in [1, 2, 3, 4]:
        sub = panel[panel["exam_history_q"] == q].copy()
        fits[f"Q{q}"] = pf.feols(
            "denial_gap ~ post_exam + log_pop + median_income + unemp_rate | year + county",
            data=sub, vcov={"CRV1": "county"})
    return fits

def emit(fits: dict, path: Path, title: str, label: str) -> None:
    """Write the etable output to disk as a LaTeX fragment."""
    tex = pf.etable(
        list(fits.values()),
        type="tex",
        coef_fmt="b (se)",
        signif_code=[0.01, 0.05, 0.1],
        labels={"post_exam": "Post-examination",
                "log_pop":   "log(Population)",
                "median_income": "Median income",
                "unemp_rate": "Unemployment rate"},
        notes=f"Standard errors clustered by county. Seed={SEED}.",
        caption=title,
        label=label,
    )
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_text(tex)

def main(panel_path: Path, out_dir: Path) -> None:
    panel = pd.read_parquet(panel_path)
    np.random.seed(SEED)

    emit(fit_table2(panel),
         out_dir / "tab2_headline.tex",
         "Effect of Intensified Examinations on the Denial Gap",
         "tab:headline")
    emit(fit_table3(panel),
         out_dir / "tab3_heterogeneity.tex",
         "Heterogeneity by Examination-History Quartile",
         "tab:heterogeneity")

if __name__ == "__main__":
    ap = argparse.ArgumentParser()
    ap.add_argument("--panel", type=Path, required=True)
    ap.add_argument("--out-dir", type=Path, required=True)
    args = ap.parse_args()
    main(args.panel, args.out_dir)
```

Notice four discipline moves. **First**, every regression specification is named and traceable to the manuscript section it appears in. **Second**, the `vcov={"CRV1": "county"}` argument is *named* — the same SE flavor named in the abstract's S3. **Third**, the `labels` dictionary maps panel column names to display names ("Post-examination" not "post_exam"), the Chapter 11.2 typography rule made executable. **Fourth**, the seed is set at module load and printed in the table notes, so the table documents the run that produced it.

### 2.2 Wire the table generator into the manuscript

In `manuscript/sections/results.tex`, `\input{}` the generated table:

```latex
\section{Results}
\label{sec:results}

The headline effect appears in Table~\ref{tab:headline}. Column (4), with year and county fixed effects and the four standard controls, identifies a 3.1 percentage-point reduction in the Black--White denial gap in covered counties post-examination, statistically significant at the 1\% level with standard errors clustered at the county level.

\input{manuscript/tables/tab2_headline.tex}

Heterogeneity by examination-history quartile appears in Table~\ref{tab:heterogeneity}. The effect is monotone-decreasing in lender finding history: lenders with the highest pre-period finding rates show the largest post-examination reductions, consistent with the targeted-scrutiny mechanism the introduction names.

\input{manuscript/tables/tab3_heterogeneity.tex}
```

Notice the discipline: the 3.1pp in the prose matches Table 2 column (4) because both come from the same fit. If a coauthor later re-runs with a different sample restriction and the coefficient changes to 2.7pp, Table 2 updates automatically but the prose still says 3.1pp — and the build will not catch this. The Chapter 11.2 fix is to `\input{}` magnitudes from a tiny `magnitudes.tex` the table generator also writes, so the prose reads `\input{manuscript/tables/headline_magnitude.tex}\,pp reduction`. Add this on the second iteration; for now, discipline magnitudes by grep'ing the manuscript before submission.

## Part 3 — Figures from matplotlib, with consistent typography

The Chapter 11.3 discipline was: every figure looks like it came from the same paper. Lab 11 enforces this through a `src/build/figstyle.py` module that sets the matplotlib rc parameters once, and a `src/build/figures.py` module that produces every figure using those settings.

### 3.1 The shared `figstyle.py`

```python
"""src/build/figstyle.py — typography settings shared across all figures."""
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

def apply():
    plt.rcParams.update({
        "font.family": "serif",
        "font.serif": ["Computer Modern Roman", "Times", "DejaVu Serif"],
        "font.size": 10,
        "axes.titlesize": 11,
        "axes.labelsize": 10,
        "xtick.labelsize": 9,
        "ytick.labelsize": 9,
        "legend.fontsize": 9,
        "figure.figsize": (6.0, 3.75),   # AEA single-column width in inches
        "figure.dpi": 300,
        "savefig.dpi": 300,
        "savefig.bbox": "tight",
        "lines.linewidth": 1.5,
        "axes.spines.top": False,
        "axes.spines.right": False,
    })

# A 4-color palette that prints well in grayscale: anchor, comparison,
# null-line, and a fourth for heterogeneity. Tested against the Lab 9
# colorblindness checklist.
PALETTE = {
    "treated":   "#1f77b4",
    "control":   "#d62728",
    "null":      "#7f7f7f",
    "highlight": "#2ca02c",
}
```

The font choice (`Computer Modern Roman`) matches the AEA template body text, so figures embedded in the manuscript do not "switch fonts" relative to surrounding prose. The 6.0-inch width matches the single-column AEA width; figures will fill the column without scaling, which preserves the typographic sizes you set. The palette is shared so the headline color ("treated") is the same across every figure in the paper — a small consistency move that signals professionalism to a referee.

## Part 4 — BibTeX validated against the Week-7 verified-bibs list

CONVENTIONS §6 requires that every empirical claim cite at least one primary source with full bibliographic info, and that fabricated citations are tagged `[CHECK]` rather than passed off as verified. Week 7 produced a master `verified.bib` file containing only citations that had been confirmed against journal websites or NBER working-paper landing pages. Lab 11 enforces the discipline by failing the build if any citation key the manuscript uses is missing from `verified.bib`, or if `verified.bib` contains any `[CHECK]` tags.

### 4.1 The `bibcheck.py` script

```python
"""src/build/bibcheck.py — fail build if any \\cite{key} is unverified."""
import re
import sys
from pathlib import Path

def cited_keys(manuscript_dir: Path) -> set[str]:
    """Scan all .tex files for \\cite{key1,key2} patterns."""
    keys = set()
    pattern = re.compile(r"\\cite[a-z]*\{([^}]+)\}")
    for tex in manuscript_dir.rglob("*.tex"):
        for match in pattern.findall(tex.read_text()):
            for key in match.split(","):
                keys.add(key.strip())
    return keys

def verified_keys(bib_path: Path) -> tuple[set[str], list[str]]:
    """Parse the .bib file; return verified keys plus any [CHECK]-tagged entries."""
    text = bib_path.read_text()
    keys = set(re.findall(r"^@\w+\{([^,]+),", text, re.MULTILINE))
    check_tags = []
    for entry in re.split(r"^@", text, flags=re.MULTILINE):
        if "[CHECK]" in entry:
            key_match = re.match(r"\w+\{([^,]+),", entry)
            if key_match:
                check_tags.append(key_match.group(1))
    return keys, check_tags

def main(manuscript_dir: Path, bib_path: Path) -> int:
    cited = cited_keys(manuscript_dir)
    verified, checks = verified_keys(bib_path)

    missing = cited - verified
    if missing:
        print(f"FAIL: {len(missing)} citation keys are not in {bib_path}:")
        for k in sorted(missing):
            print(f"  - {k}")
        return 1
    if checks:
        print(f"FAIL: {len(checks)} [CHECK]-tagged entries in {bib_path}:")
        for k in checks:
            print(f"  - {k}")
        return 2

    print(f"OK: {len(cited)} citations all verified.")
    return 0

if __name__ == "__main__":
    md = Path(sys.argv[1])
    bp = Path(sys.argv[2])
    sys.exit(main(md, bp))
```

The script is intentionally strict. A missing citation key is a build failure, and so is a `[CHECK]`-tagged entry that survived into `verified.bib`: by CONVENTIONS §6, the verified bib should contain only confirmed entries. Keep in-progress citations in a separate `unverified.bib` the build does not read.

```makefile
        $(PANEL) src/build/tables.py
	$(PYTHON) -m src.build.tables --panel $(PANEL) --out-dir manuscript/tables

manuscript/figures/fig1_event_study.pdf manuscript/figures/fig2_spec_curve.pdf: \
        $(PANEL) src/build/figures.py src/build/figstyle.py
	$(PYTHON) -m src.build.figures --panel $(PANEL) --out-dir manuscript/figures

bibcheck: manuscript/bibliography/verified.bib
	$(PYTHON) -m src.build.bibcheck manuscript manuscript/bibliography/verified.bib

stamp:
	@echo "git:$$(git rev-parse --short HEAD)" > manuscript/build_stamp.tex
	@date +%Y-%m-%d > manuscript/build_date.tex

paper_clean:
	rm -f manuscript/main.pdf manuscript/main.aux manuscript/main.bbl \
	      manuscript/main.blg manuscript/main.log manuscript/main.out
	rm -f manuscript/tables/*.tex manuscript/figures/*.pdf
	rm -f manuscript/build_stamp.tex manuscript/build_date.tex
```

The two-pass-plus-bibtex sequence is the standard LaTeX dance: the first pass writes the `.aux` with citation labels, `bibtex` resolves them into a `.bbl`, the second pass reads the `.bbl`, and the third resolves cross-references. Skipping the third pass leaves "??" in cross-references — a common bug.

The `| bibcheck` order-only prerequisite runs `bibcheck` *before* `pdflatex`, so if any citation is unverified the build fails before LaTeX sees the manuscript.

### 5.1 Run it

```bash
make stamp           # write the build stamp
make tables          # ~5 seconds
make figures         # ~10 seconds
make bibcheck        # ~1 second
make paper           # ~30 seconds (three LaTeX passes + bibtex)
open manuscript/main.pdf    # macOS; use xdg-open on Linux
```

If the build succeeds, open the PDF and check three things visually. **First**, the column widths and font sizes match the AEA template (compare against `manuscript/aea/aea-template.tex` compiled separately). **Second**, every table and figure appears where the manuscript expects it, with the caption number matching the `\ref{}` in the prose. **Third**, the bibliography on the last page has every cited paper, formatted in AEA style.

### 5.2 The byte-reproducibility check

The final discipline: `make clean && make paper` produces a PDF byte-identical to a previous run, *modulo* the timestamp in `build_date.tex` and the `/CreationDate` PDF metadata field. Two SHA-256s may differ because `pdflatex` embeds a `/CreationDate` timestamp. To get byte-exact reproducibility, set `SOURCE_DATE_EPOCH` before the LaTeX call:

```bash
SOURCE_DATE_EPOCH=$(git log -1 --format=%ct) make clean paper
```

With `SOURCE_DATE_EPOCH` pinned, two builds produce byte-identical PDFs (the same trick Debian uses for reproducible builds). For capstone purposes, content-level reproducibility is sufficient; byte-level is a stretch goal.

## Part 6 — The build manifest

Stamp the PDF with a manifest naming the commit, the seed, the panel SHA-256, and the build timestamp. The Chapter 11.4 logic is: a future reader (the referee, the discussant, future-you in 2030) needs to know *exactly* which version of the code produced the PDF they are holding.

### 6.1 The `stamp.py` module

```python
"""src/build/stamp.py — write the build manifest as a LaTeX-includable fragment."""
import hashlib
import subprocess
from pathlib import Path

def main(panel_path: Path, out_path: Path) -> None:
    sha = subprocess.check_output(["git", "rev-parse", "--short", "HEAD"]).decode().strip()
    panel_sha = hashlib.sha256(panel_path.read_bytes()).hexdigest()[:12]
    timestamp = subprocess.check_output(["date", "+%Y-%m-%d %H:%M:%S"]).decode().strip()
    text = (
        f"git={sha}\\quad panel-sha256={panel_sha}\\quad "
        f"seed=42\\quad built={timestamp}"
    )
    out_path.write_text(text)

if __name__ == "__main__":
    import sys
    main(Path(sys.argv[1]), Path(sys.argv[2]))
```

In `main.tex`, the `\thanks{}` macro `\input{}`s this fragment, so every PDF carries the manifest on the first-page footnote. A referee printing and reading offline always has the git hash; they can clone at that commit and reproduce. Leah's patent-text-analysis capstone extends the pattern with a `spacy_version` field (spaCy's tokenization can drift across releases). The manifest grows with the project's reproducibility needs.

## Part 7 — Common failure modes

Five failures to anticipate, with their fixes.

**Failure 1: `pdflatex` runs but the PDF has "?" in citations.** You forgot the `bibtex` pass or did not run `pdflatex` enough times after `bibtex`. The fix is in the Makefile rule above: always three `pdflatex` passes with `bibtex` between the first and second. If you ran by hand and got "?", do `bibtex main && pdflatex main && pdflatex main` to resolve.

**Failure 2: `bibcheck` fails on a citation you swear is verified.** Open `manuscript/bibliography/verified.bib` and grep for the key. If it is there, check for whitespace in the key (`\cite{Gao Sun 2019}` vs. `\cite{GaoSun2019}` — LaTeX cares). If it is not there, you have a real missing citation; add it from a primary source, or — if you cannot yet verify — remove the `\cite{}` from the manuscript and add a `[CHECK]`-tagged comment as a placeholder, per CONVENTIONS §6.

**Failure 3: `pyfixest.etable()` produces a table that is too wide for the column.** Three fixes in order of preference. (a) Reduce the number of columns — four model-columns plus a row-label column is the AEA-template ceiling for single-column. (b) Shorten the variable display labels in the `labels=` dictionary. (c) Pass `type="tex_no_resize"` and add a `\resizebox{\columnwidth}{!}{...}` wrapper in the `.tex` fragment — but only if (a) and (b) cannot get you there, because `\resizebox` shrinks the fonts and breaks the typography consistency that figstyle.py is enforcing.

**Failure 4: Figures appear in the PDF but with the wrong font.** Two causes. Either you forgot to call `apply()` at the top of the figure function (the rcParams settings are not active and matplotlib fell back to its default sans-serif), or your LaTeX install does not have the Computer Modern fonts. Check the figure-function code first; if `apply()` is there, install the missing fonts via `sudo apt install texlive-fonts-recommended texlive-fonts-extra` (Linux) or `tlmgr install lm` (any platform with TeXLive).

**Failure 5: `make clean && make paper` succeeds but the SHA-256 differs across runs.** This is the `/CreationDate` issue from Part 5.2. Set `SOURCE_DATE_EPOCH` before the build, and confirm by running `pdfinfo manuscript/main.pdf` and checking the `CreationDate` field is identical across runs.
