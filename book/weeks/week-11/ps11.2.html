Part 1 — Estimate the regressions and produce the raw etable() output (20 points)

The single sin this part prevents: producing a table from a single regression. Almost every headline table in empirical finance has at least three columns showing how the headline coefficient moves (or holds steady) as the specification varies. The columns are the spec menu from PS 8.1, distilled to the three or four variants the paper actually claims defend the result.

(a) Choose your column structure (5 pts). Write in build_table.py’s docstring the four (or N) specifications that will become the columns of Table 2. The canonical structure for a DiD paper:

  • Column (1): Baseline — your headline interaction term, with the minimum fixed-effects saturation (unit + time).
  • Column (2): Add one more layer of saturation (unit + time + controls).
  • Column (3): The preferred saturation — unit × interacted-time fixed effects or the equivalent, the one the abstract’s headline number comes from.
  • Column (4): The tight sample — restrict to the cleanest subset (e.g., OCC-supervised only, balanced panel only) to show the result survives sample-restriction.

For non-DiD designs, the column structure varies. An IV paper typically uses: (1) OLS, (2) reduced-form, (3) first-stage F-stat anchor, (4) 2SLS with controls. An event study typically uses: (1) market-model abnormal returns, (2) Fama-French three-factor abnormal returns, (3) Carhart four-factor, (4) the peer-firm placebo. State the structure in the docstring with one sentence per column.

(b) Estimate the regressions using pyfixest.feols() (8 pts). Code the estimation. Cluster the standard errors at the level your design dictates — county for Maya, firm for Devon and Sam, census-tract for Priya, four-digit IPC for Leah — the same cluster level as Table 1 and the abstract. The estimation is verified for the textbook example (Maya):

# build_table.py — Maya's four-column DiD estimation, verified
import numpy as np
import pandas as pd
import pyfixest as pf

rng = np.random.default_rng(42)
# Simulated HMDA-like data: 1000 counties × 7 years; treatment in 2017 for 614 of them.
N_county, T = 1000, 7
years = np.arange(2014, 2021)
treated = rng.choice(N_county, 614, replace=False)
panel = pd.DataFrame({
    "county": np.repeat(np.arange(N_county), T),
    "year": np.tile(years, N_county),
})
panel["treated"] = panel["county"].isin(treated).astype(int)
panel["post"] = (panel["year"] >= 2017).astype(int)
panel["did"] = panel["treated"] * panel["post"]
# True ATT: -1.4 pp on denial gap; with noise.
panel["denial_gap"] = (
    6.1
    - 1.4 * panel["did"]
    + 0.5 * (panel["year"] - 2017)
    + rng.normal(0, 1.5, len(panel))
)
panel["lender_area"] = panel["county"] % 50
panel["lender_year"] = panel["lender_area"].astype(str) + "_" + panel["year"].astype(str)
m1 = pf.feols("denial_gap ~ did | county + year", data=panel, vcov={"CRV1": "county"})
m2 = pf.feols("denial_gap ~ did | county + year + lender_area", data=panel, vcov={"CRV1": "county"})
m3 = pf.feols("denial_gap ~ did | county + year + lender_year", data=panel, vcov={"CRV1": "county"})
m4 = pf.feols(
    "denial_gap ~ did | county + year",
    data=panel.query("treated == 1 or year >= 2016"),
    vcov={"CRV1": "county"},
)
for i, m in enumerate([m1, m2, m3, m4], start=1):
    print(f"m{i} did: {m.coef()['did']:.3f} ({m.se()['did']:.3f}); N = {m._N}")

Verified arithmetic with seed=42, 1000 simulated counties, true ATT = -1.4 pp, noise SD = 1.5:

m1 did: -1.383 (0.074); N = 7000
m2 did: -1.383 (0.074); N = 7000
m3 did: -1.361 (0.077); N = 7000
m4 did: -1.391 (0.107); N = 6228

The four coefficients land in the -1.36 to -1.39 band; the cluster-robust SEs are 0.07 to 0.11. The cross-column stability is the point of the table — the headline number does not move materially as we add saturation or restrict the sample. (The SEs are tight in this simulation because the panel is large and the noise process is i.i.d.; real-world panels with serial correlation will produce SEs an order of magnitude larger, which is why the cluster level matters.)

(c) Render etable() to LaTeX (7 pts). Use pyfixest.etable() with the parameters that produce publication-clean output:

table_latex = pf.etable(
    [m1, m2, m3, m4],
    type="tex",
    coef_fmt="b (se)",  # coefficient and SE only (no t-statistic)
    digits=3,
    keep=["did"],  # show only the headline coefficient row
    custom_model_names={"m1": "(1) Baseline", "m2": "(2) + lender FE",
                        "m3": "(3) Preferred", "m4": "(4) Tight sample"},
    notes=(
        "Standard errors clustered at the county level in parentheses. "
        "*** p<0.01, ** p<0.05, * p<0.10. The 2017 fair-lending examination "
        "intensification is identified via staggered rollout across "
        "OCC-supervised counties."
    ),
)
with open("table-main.tex", "w") as f:
    f.write(table_latex)

The output is a fragment — \begin{tabular}{lcccc} \toprule … \bottomrule \end{tabular} — wrap it in \begin{table}[H] \centering \caption{…} \label{tab:main} … \end{table} in your manuscript or in the standalone table-main.tex. Compile to PDF with pdflatex table-main.tex (after ensuring booktabs and caption are loaded).

Part 2 — The four design choices: rounding, saturation indicators, sample footer, notes (20 points)

The single sin this part prevents: letting the defaults decide your typography. pyfixest.etable() has sensible defaults; you will override several of them deliberately.

(a) Rounding discipline (5 pts). Three decimals for coefficients, two for R-squared, integer for observations and clusters. Justify in footnote-anatomy.md: why three decimals for coefficients? Because the cluster-robust SE for your headline cell is typically in the second decimal (0.31 in Maya’s case), and reporting fewer decimals than the SE allows is information loss — the reader cannot reconstruct the t-statistic. Why not four decimals? Because the fourth decimal of your coefficient is below the noise floor of your SE — reporting it is false precision and a careful referee will notice. The convention “report one more decimal than your SE supports” is the rule; for a 0.31 SE, report three coefficient decimals; for a 0.03 SE, report four.

(b) Fixed-effects indicator block (5 pts). A block of Yes/No rows showing, for each model, which fixed effects are saturated. The block goes below the coefficient row and above the sample-size row. For Maya’s four columns:

(1) (2) (3) (4)
County FE Yes Yes – Yes
Year FE Yes Yes – Yes
County × Year FE – – Yes –
Treated × Year FE – Yes – –
Cluster: county Yes Yes Yes Yes

The two-character indicator “Yes” / “–” (with em-dash for “no” not “No,” which reads as redundant against “Yes”) signals saturation. The block tells the reader, in one glance, which columns saturate what — answering Question 4 of the one-glance test (“what controls and fixed effects were in?”).

(c) Sample footer (5 pts). Two rows immediately below the FE block:

Observations | 7,000 | 7,000 | 7,000 | 6,228 |
Counties | 1,000 | 1,000 | 1,000 | 1,000 |

The integer-formatted observation count and the cluster count answer Question 3 of the one-glance test (“what is the sample?”). The drop from 7,000 to 6,228 in column (4) signals the tight-sample restriction (the treated == 1 or year >= 2016 filter retains all treated-county years and untreated-county years from 2016 onward); the reader sees the cost of the restriction immediately. If you omit the cluster count and report only observations, the reader cannot judge the clustering and the table fails this one-glance question.

(d) Notes footer (5 pts). A single paragraph below the bottom rule, 3–5 sentences:

Notes: This table reports staggered difference-in-differences estimates of the 2017 OCC fair-lending examination intensification on the county-level Black–White denial-rate gap. The unit of observation is the county-year; the sample covers all U.S. counties with at least 50 home-purchase loan applications per year, 2014–2020 (N = 7,000 county-years for columns 1–3; the tight sample in column 4 excludes never-treated counties and pre-2016 observations). Standard errors clustered at the county level in parentheses. The headline coefficient is the interaction Treated × Post, capturing the average treatment effect on the treated. *** p<0.01, ** p<0.05, * p<0.10.

The notes do five jobs: (i) describe the table; (ii) name the unit of observation; (iii) define the sample; (iv) state the clustering; (v) define the stars. Reorder freely, but include all five. A notes-line that says “see paper for details” is a fail; the notes must make the table self-contained against a reader who has not read your paper.

Part 3 — Stars discipline and the multiple-testing footnote (20 points)

The single sin this part prevents: reporting stars from naive p-values when your design has been multiple-tested. Chapter 10.2 and PS 10.2 established that when you test many hypotheses (the four columns of your table, the heterogeneity cuts of your table, the placebo events of your design), the naive p-values overstate significance. The stars on your table should reflect that — either by displaying multiple-testing-adjusted stars, or by reporting naive stars with a footnote that explicitly references the adjusted column in a supplementary table.

(a) Decide your stars convention (5 pts). Three choices, ranked from most-rigorous to most-conventional:

  1. Bonferroni / BH-FDR / Romano-Wolf stars in the main table. Report stars only against the adjusted p-value. Use a footnote: “Stars reflect Romano-Wolf step-down stepwise stepwise stepwise stepdown adjusted p-values; naive p-values reported in Appendix Table A-2.” This is the most defensible choice for a paper with \geq 4 tested cells.
  2. Naive stars in the main table, with adjusted column in a parallel appendix table. Most common. Report stars against naive p-values; in the footnote, state: “Naive p-values reported; Bonferroni-adjusted p-values across the four columns and three heterogeneity cuts reported in Appendix Table A-2.” The reader can verify the adjusted column.
  3. Naive stars without disclosure. Common in older papers; not defensible at modern top journals when the paper makes claims across multiple tested cells.

Pick (1) or (2). State your choice in stars-discipline.md with one sentence of justification. If you pick (2), include the appendix table reference in the main-table notes.

(b) The three stars or full p-values? (5 pts). The convention in finance is three stars at p<0.01, p<0.05, p<0.10. Some methodological papers prefer reporting full p-values (e.g., p=0.023). Stars are easier for the one-glance test; full p-values are more information-dense. The compromise is stars in the main table, full p-values in the appendix. State which you use and justify.

(c) The p<0.10 star is contentious — defend it or drop it (5 pts). The p<0.10 single-star convention is increasingly criticized as “p-hacking-friendly” because authors can frame marginally-significant results as “suggestive” while displaying a star. Two defensible positions: (i) keep the single star, but only report it when the result is consistent across columns or aligns with a pre-registered prediction; (ii) drop the single star entirely and report only p<0.05 and p<0.01 stars. State your position in one paragraph (~75 words).

(d) The footnote that links to PS 10.2 (5 pts). Add one sentence to the table notes: “Family-wise error rate across the four columns and the three heterogeneity cuts is controlled at \alpha = 0.05 using the [Bonferroni / BH-FDR / Romano-Wolf] procedure; see Appendix Table A-2 and §5.3 of the paper.” This sentence is the disclosure the careful referee will check; without it, the reader cannot verify that the multiple-testing adjustment from PS 10.2 was applied to the main table.

Part 4 — The one-glance-test memo (20 points)

This is the memo that defends Table 2 against the four questions from Chapter 11.2 §1. Write it in one-glance.md, ~400 words.

(a) Answer each of the four questions in writing (12 pts, 3 each). Write a one-paragraph answer to each:

  1. What is the headline coefficient? Name the cell: “row labeled did, column 3 (Preferred).” State the magnitude: “-1.41 percentage points.” State the unit: “the unit is percentage points of the county-level Black-White denial-rate gap, ranging from 0 to ~10pp in the sample.” A reader should be able to locate, read, and interpret that cell in 10 seconds.
  2. What is its precision? Name the cell: “the parenthesized 0.31 in column 3 is the cluster-robust standard error.” State the implied t-statistic: -1.41 / 0.31 = -4.55. State the implied 95% CI: (-2.02, -0.80).
  3. What is the sample? Name the cell: “7,000 county-years across 1,000 counties (columns 1–3); 6,228 county-years across 1,000 counties in the tight column 4.” State the time window: 2014–2020. State the unit of observation: county-year.
  4. What controls and fixed effects were in? Name the FE block: “column 3 saturates with county × year fixed effects, the most demanding saturation. Standard errors clustered at the county level in every column.” State what is not in: “no time-varying control variables enter directly; the saturation absorbs them.”

(b) The sixty-second test (5 pts). Have a peer who has not read your paper read Table 2 for 60 seconds (literally — set a timer). Then ask them the four questions. Record their answers in one-glance.md. If they answered all four correctly, the table passes. If they missed one or two, the table needs revision at the corresponding row/footnote — and you note the revision in the memo.

(c) What you stripped (3 pts). List in the memo the rows, columns, or coefficients you considered including but stripped. Common strips: (i) control-variable coefficients (banished to an appendix or omitted entirely), (ii) the constant term (rarely useful in a fixed-effects regression), (iii) R-squared and adjusted R-squared (the FE saturation absorbs so much variance that R² is mechanically high and uninformative — many top journals omit it from fixed-effects tables), (iv) F-statistics for joint tests (move to an appendix unless they are central). For each strip, one sentence of justification.

Part 5 — Render to PDF and compile the camera-ready (20 points)

(a) The compiled PDF (10 pts). Compile table-main.tex to table-main.pdf. The PDF should be one page, with the caption above the table, the body in the middle, the notes below. Use \begin{table}[H] \centering \caption{Headline DiD estimates: 2017 OCC fair-lending examination intensification reduces the Black-White denial-rate gap.} \label{tab:main} to wrap. The caption is informative — it tells the reader the headline finding without making them read the body. Avoid the generic “Main results.”

(b) Check the typography (5 pts). Three checks:

  1. Booktabs rules. Top, middle, and bottom rules are \toprule, \midrule, \bottomrule — not \hline. Vertical lines (|) are forbidden. If you see \hline or | in your LaTeX, replace.
  2. Decimal alignment. In LaTeX, columns of numbers should align on the decimal point. pyfixest.etable() handles this automatically; verify visually by viewing the PDF.
  3. Caption above, notes below. Captions go above tables (the convention; figures get captions below). Notes go below the bottom rule, in a slightly smaller font (often \footnotesize or \small).

(c) The fit-on-one-page check (5 pts). The table must fit on a single 8.5×11 page with one-inch margins, with the caption and notes. If it does not, either reduce the columns (consider whether you really need a fifth column) or break into a panel structure (Panel A: pooled; Panel B: by subgroup). State in one-glance.md whether the table fit on first compile or whether you reduced. A table that requires two pages is a sign that the column structure is over-engineered — strip back.

```python
panel["treated"] = panel["county"].isin(treated).astype(int)
panel["post"] = (panel["year"] >= 2017).astype(int)
panel["did"] = panel["treated"] * panel["post"]
# True ATT: -1.4 pp on denial gap; with noise.
panel["denial_gap"] = (
    6.1
    - 1.4 * panel["did"]
    + 0.5 * (panel["year"] - 2017)
    + rng.normal(0, 1.5, len(panel))
)
panel["lender_area"] = panel["county"] % 50
panel["lender_year"] = panel["lender_area"].astype(str) + "_" + panel["year"].astype(str)
m1 = pf.feols("denial_gap ~ did | county + year", data=panel, vcov={"CRV1": "county"})
m2 = pf.feols("denial_gap ~ did | county + year + lender_area", data=panel, vcov={"CRV1": "county"})
m3 = pf.feols("denial_gap ~ did | county + year + lender_year", data=panel, vcov={"CRV1": "county"})
m4 = pf.feols(
    "denial_gap ~ did | county + year",
    data=panel.query("treated == 1 or year >= 2016"),
    vcov={"CRV1": "county"},
)
for i, m in enumerate([m1, m2, m3, m4], start=1):
    print(f"m{i} did: {m.coef()['did']:.3f} ({m.se()['did']:.3f}); N = {m._N}")
```

Verified arithmetic with `seed=42`, 1000 simulated counties, true ATT = $-1.4$ pp, noise SD = 1.5:

```
m1 did: -1.383 (0.074); N = 7000
m2 did: -1.383 (0.074); N = 7000
m3 did: -1.361 (0.077); N = 7000
m4 did: -1.391 (0.107); N = 6228
```

The four coefficients land in the $-1.36$ to $-1.39$ band; the cluster-robust SEs are $0.07$ to $0.11$. The cross-column stability is the point of the table — the headline number does not move materially as we add saturation or restrict the sample. (The SEs are tight in this simulation because the panel is large and the noise process is i.i.d.; real-world panels with serial correlation will produce SEs an order of magnitude larger, which is why the cluster level matters.)

## Part 2 — The four design choices: rounding, saturation indicators, sample footer, notes (20 points)

The single sin this part prevents: **letting the defaults decide your typography.** `pyfixest.etable()` has sensible defaults; you will override several of them deliberately.

**(a) Rounding discipline (5 pts).** Three decimals for coefficients, two for R-squared, integer for observations and clusters. Justify in `footnote-anatomy.md`: *why three decimals for coefficients?* Because the cluster-robust SE for your headline cell is typically in the second decimal (0.31 in Maya's case), and reporting fewer decimals than the SE allows is *information loss* — the reader cannot reconstruct the t-statistic. *Why not four decimals?* Because the fourth decimal of your coefficient is below the noise floor of your SE — reporting it is *false precision* and a careful referee will notice. The convention "report one more decimal than your SE supports" is the rule; for a 0.31 SE, report three coefficient decimals; for a 0.03 SE, report four.

**(b) Fixed-effects indicator block (5 pts).** A block of Yes/No rows showing, for each model, which fixed effects are saturated. The block goes *below* the coefficient row and *above* the sample-size row. For Maya's four columns:

| | (1) | (2) | (3) | (4) |
|---|---|---|---|---|
| County FE | Yes | Yes | – | Yes |
| Year FE | Yes | Yes | – | Yes |
| County × Year FE | – | – | Yes | – |
| Treated × Year FE | – | Yes | – | – |
| Cluster: county | Yes | Yes | Yes | Yes |

The two-character indicator "Yes" / "–" (with em-dash for "no" not "No," which reads as redundant against "Yes") signals saturation. The block tells the reader, in one glance, which columns saturate what — answering Question 4 of the one-glance test ("what controls and fixed effects were in?").

**(c) Sample footer (5 pts).** Two rows immediately below the FE block:

| Observations | 7,000 | 7,000 | 7,000 | 6,228 |
| Counties | 1,000 | 1,000 | 1,000 | 1,000 |

The integer-formatted observation count and the cluster count answer Question 3 of the one-glance test ("what is the sample?"). The drop from 7,000 to 6,228 in column (4) signals the tight-sample restriction (the `treated == 1 or year >= 2016` filter retains all treated-county years and untreated-county years from 2016 onward); the reader sees the cost of the restriction immediately. If you omit the cluster count and report only observations, the reader cannot judge the clustering and the table fails this one-glance question.

**(d) Notes footer (5 pts).** A single paragraph below the bottom rule, 3–5 sentences:

> *Notes: This table reports staggered difference-in-differences estimates of the 2017 OCC fair-lending examination intensification on the county-level Black–White denial-rate gap. The unit of observation is the county-year; the sample covers all U.S. counties with at least 50 home-purchase loan applications per year, 2014–2020 (N = 7,000 county-years for columns 1–3; the tight sample in column 4 excludes never-treated counties and pre-2016 observations). Standard errors clustered at the county level in parentheses. The headline coefficient is the interaction Treated × Post, capturing the average treatment effect on the treated. \*\*\* p<0.01, \*\* p<0.05, \* p<0.10.*

The notes do five jobs: (i) describe the table; (ii) name the unit of observation; (iii) define the sample; (iv) state the clustering; (v) define the stars. Reorder freely, but include all five. A notes-line that says "see paper for details" is a fail; the notes must make the table self-contained against a reader who has not read your paper.

## Part 3 — Stars discipline and the multiple-testing footnote (20 points)

The single sin this part prevents: **reporting stars from naive p-values when your design has been multiple-tested.** Chapter 10.2 and PS 10.2 established that when you test many hypotheses (the four columns of your table, the heterogeneity cuts of your table, the placebo events of your design), the naive p-values overstate significance. The stars on your table should reflect that — either by displaying multiple-testing-adjusted stars, or by reporting naive stars with a footnote that explicitly references the adjusted column in a supplementary table.

**(a) Decide your stars convention (5 pts).** Three choices, ranked from most-rigorous to most-conventional:

1. *Bonferroni / BH-FDR / Romano-Wolf stars in the main table.* Report stars only against the adjusted p-value. Use a footnote: *"Stars reflect Romano-Wolf step-down stepwise stepwise stepwise stepdown adjusted p-values; naive p-values reported in Appendix Table A-2."* This is the most defensible choice for a paper with $\geq 4$ tested cells.
2. *Naive stars in the main table, with adjusted column in a parallel appendix table.* Most common. Report stars against naive p-values; in the footnote, state: *"Naive p-values reported; Bonferroni-adjusted p-values across the four columns and three heterogeneity cuts reported in Appendix Table A-2."* The reader can verify the adjusted column.
3. *Naive stars without disclosure.* Common in older papers; *not* defensible at modern top journals when the paper makes claims across multiple tested cells.

Pick (1) or (2). State your choice in `stars-discipline.md` with one sentence of justification. If you pick (2), include the appendix table reference in the main-table notes.

**(b) The three stars or full p-values? (5 pts).** The convention in finance is three stars at $p<0.01, p<0.05, p<0.10$. Some methodological papers prefer reporting full p-values (e.g., $p=0.023$). Stars are easier for the one-glance test; full p-values are more information-dense. The compromise is *stars in the main table, full p-values in the appendix*. State which you use and justify.

**(c) The $p<0.10$ star is contentious — defend it or drop it (5 pts).** The $p<0.10$ single-star convention is increasingly criticized as "p-hacking-friendly" because authors can frame marginally-significant results as "suggestive" while displaying a star. Two defensible positions: (i) keep the single star, but only report it when the result is consistent across columns or aligns with a pre-registered prediction; (ii) drop the single star entirely and report only $p<0.05$ and $p<0.01$ stars. State your position in one paragraph (~75 words).

**(d) The footnote that links to PS 10.2 (5 pts).** Add one sentence to the table notes: *"Family-wise error rate across the four columns and the three heterogeneity cuts is controlled at $\alpha = 0.05$ using the [Bonferroni / BH-FDR / Romano-Wolf] procedure; see Appendix Table A-2 and §5.3 of the paper."* This sentence is the disclosure the careful referee will check; without it, the reader cannot verify that the multiple-testing adjustment from PS 10.2 was applied to the main table.

## Part 4 — The one-glance-test memo (20 points)

This is the memo that defends Table 2 against the four questions from Chapter 11.2 §1. Write it in `one-glance.md`, ~400 words.

**(a) Answer each of the four questions in writing (12 pts, 3 each).** Write a one-paragraph answer to each:

1. *What is the headline coefficient?* Name the cell: "row labeled `did`, column 3 (Preferred)." State the magnitude: "$-1.41$ percentage points." State the unit: "the unit is percentage points of the county-level Black-White denial-rate gap, ranging from 0 to ~10pp in the sample." A reader should be able to locate, read, and interpret that cell in 10 seconds.
2. *What is its precision?* Name the cell: "the parenthesized 0.31 in column 3 is the cluster-robust standard error." State the implied t-statistic: $-1.41 / 0.31 = -4.55$. State the implied 95% CI: $(-2.02, -0.80)$.
3. *What is the sample?* Name the cell: "7,000 county-years across 1,000 counties (columns 1–3); 6,228 county-years across 1,000 counties in the tight column 4." State the time window: 2014–2020. State the unit of observation: county-year.
4. *What controls and fixed effects were in?* Name the FE block: "column 3 saturates with county × year fixed effects, the most demanding saturation. Standard errors clustered at the county level in every column." State what is *not* in: "no time-varying control variables enter directly; the saturation absorbs them."

**(b) The sixty-second test (5 pts).** Have a peer who has not read your paper read Table 2 for 60 seconds (literally — set a timer). Then ask them the four questions. Record their answers in `one-glance.md`. If they answered all four correctly, the table passes. If they missed one or two, the table needs revision at the corresponding row/footnote — and you note the revision in the memo.

**(c) What you stripped (3 pts).** List in the memo the rows, columns, or coefficients you considered including but stripped. Common strips: (i) control-variable coefficients (banished to an appendix or omitted entirely), (ii) the constant term (rarely useful in a fixed-effects regression), (iii) R-squared and adjusted R-squared (the FE saturation absorbs so much variance that R² is mechanically high and uninformative — many top journals omit it from fixed-effects tables), (iv) F-statistics for joint tests (move to an appendix unless they are central). For each strip, one sentence of justification.

## Part 5 — Render to PDF and compile the camera-ready (20 points)

**(a) The compiled PDF (10 pts).** Compile `table-main.tex` to `table-main.pdf`. The PDF should be one page, with the caption above the table, the body in the middle, the notes below. Use `\begin{table}[H] \centering \caption{Headline DiD estimates: 2017 OCC fair-lending examination intensification reduces the Black-White denial-rate gap.} \label{tab:main}` to wrap. The caption is *informative* — it tells the reader the headline finding without making them read the body. Avoid the generic *"Main results."*

**(b) Check the typography (5 pts).** Three checks:

1. *Booktabs rules.* Top, middle, and bottom rules are `\toprule`, `\midrule`, `\bottomrule` — *not* `\hline`. Vertical lines (`|`) are forbidden. If you see `\hline` or `|` in your LaTeX, replace.
2. *Decimal alignment.* In LaTeX, columns of numbers should align on the decimal point. `pyfixest.etable()` handles this automatically; verify visually by viewing the PDF.
3. *Caption above, notes below.* Captions go *above* tables (the convention; figures get captions below). Notes go below the bottom rule, in a slightly smaller font (often `\footnotesize` or `\small`).

**(c) The fit-on-one-page check (5 pts).** The table must fit on a single 8.5×11 page with one-inch margins, with the caption and notes. If it does not, either reduce the columns (consider whether you really need a fifth column) or break into a panel structure (Panel A: pooled; Panel B: by subgroup). State in `one-glance.md` whether the table fit on first compile or whether you reduced. A table that requires two pages is a sign that the column structure is over-engineered — strip back.
